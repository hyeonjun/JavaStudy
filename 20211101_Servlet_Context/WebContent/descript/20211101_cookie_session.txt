* 쿠키 : 
	웹 브라우저에서 서버로 어떤 데이터를 요청하면 
	서버 측은 알맞은 로직을 수행한 후, 데이터를 웹 브라우저로 응답
	서버는 웹 브라우저와의 관계를 종료
		- 관계 종료 : 연결을 끊음(http 프로토콜 특징)
		   -> 데이터를 유지할 경우 -> 웹 브라우저의 저장소인 쿠키에 저장
	쿠키는 서버에서 생성, 저장은 클라이언트 측의 웹 브라우저에 저장된다.
	쿠키 : 4kb / 300개 정도를 저장할 수 있다.
	단(주의), 웹 브라우저 내에서는 접속(요청)된 URL을 대표이름으로 쿠키를 저장하기 때문에 초기 접속 URL이 그대로 사용되어야 한다.
	
	예) http://localhost:9000/test/test.jsp
	   http://127.0.0.1:9000/test/test.jsp => 두 브라우저의 쿠키는 다르다!!

쿠키 생성 : Cookie 클래스를 이용하여 객체화
쿠키 저장 : response.addCookie(쿠키객체명);

쿠키 주요 메서드
	setMaxAge() : 쿠키 유효기간 설정(기본값은 웹 브라우저 종료 시 자동 소멸)
	setPath() : 쿠키의 유효 경로 설정 (서브 도메인 사용 시 설정)
				메인 도메인 : naver.com / 서브 도메인 : cafe.naver.com, blog.naver.com
	setValue() : 쿠키의 값을 설정 (쿠키는 수정하는 메서드가 존재하지 않는다.)
	setVersion() : 쿠키 버전 설정
	
	getMaxAge()
	getPath()
	getValue()
	getVersion()
	getName() : 쿠키 이름 추출
	
	* 쿠키는 보안상의 취약점 때문에 중요하지 않은 데이터를 동일 브라우저를 사용하는
	  웹 애플리케이션끼리 공유할 수 있는 데이터를 임시 저장할 때 사용
	  또는 회원가입화면 -> 약관 동의 화면 -> 기타 선택화면 -> 가입완료화면
	          초기데이터 --------------------------> 최종완료화면까지 필요할 경우 잠시 사용한다.

* 세션 : 서버의 임시저장소에 저장되기 때문에 쿠키처럼 시각적으로 확인이 불가능
		세션은 서버 내에서만 접근이 가능

세션 사용 방법
1. JSP: session 내장 객체가 존재 / session 내장 객체를 사용하지 않도록 설정도 가능하다

2. Servlet: getSession() 을 이용하여 세션객체를 얻어와야한다.

세션 주요 메서드
  setAttribute() : 세션에 데이터를 저장
  getAttribute() : 세션에 저장된 데이터 추출
  
  getAttributeNames() : 세션에 저장되어 있는 모든 데이터의 이름을 반환
  
  getId() : 자동으로 생성된 세션의 ID 추출
  
  isNew() : 세션이 최초로 생성되었는지에 대한 boolean 값을 반환
  
  getMaxInactiveInterval() : 세션에 대한 유요기간 추출
  
  removeAttribute() : 세션에 저장된 특정 데이터를 제거(삭제)
  
  Invalidate() : 세션의 모든 데이터를 삭제
